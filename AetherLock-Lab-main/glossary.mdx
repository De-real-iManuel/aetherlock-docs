# Glossary

## Core Protocol Terms

- **AetherLock**: Omnichain escrow protocol enabling cross-chain asset exchange with AI-powered Proof of Task Verification (PoTv)
- **PoTv (Proof of Task Verification)**: Proprietary consensus mechanism that validates task completion quality before fund release using AI analysis, zero-knowledge identity verification, and on-chain settlement
- **Escrow**: Smart contract holding funds until predefined conditions are met and verified through the PoTv mechanism
- **Omnichain**: Capability to operate across multiple blockchain networks seamlessly, enabling cross-chain asset transfers and unified user experiences

## Blockchain Platforms

- **Solana**: High-performance blockchain serving as the primary escrow layer for AetherLock, offering fast finality and low transaction costs
- **ZetaChain**: Universal blockchain platform enabling omnichain messaging and asset routing between different blockchain networks
- **Somnia**: High-throughput EVM-compatible blockchain used for settlement operations, providing fast finality and scalability
- **Sui**: Layer 1 blockchain with object-centric data model, supported for wallet connections
- **TON (The Open Network)**: Telegram-integrated blockchain network, supported for wallet connections

## Smart Contract Frameworks

- **Anchor**: Framework for Solana smart contract development using Rust, providing security primitives and simplified account management
- **Hardhat**: Ethereum development environment used for deploying and testing ZetaChain Universal App contracts
- **Foundry**: Fast, portable toolkit for Ethereum application development, used for Somnia contract deployment

## AI and Verification

- **AWS Bedrock**: Amazon's managed AI service providing access to foundation models like Claude for evidence analysis
- **Claude**: Anthropic's large language model accessed via AWS Bedrock for task completion verification
- **Arcanum**: Primary AI provider in the fallback chain for evidence analysis
- **Confidence Score**: Numerical value (0-100) representing AI certainty in task completion verification
- **Evidence Analysis Pipeline**: Multi-stage process for evaluating submitted work proof using AI models
- **Prompt Engineering**: Technique for crafting effective AI prompts to maximize verification accuracy

## Identity and Privacy

- **zkMe**: Zero-knowledge identity verification provider offering privacy-preserving KYC (Know Your Customer)
- **Zero-Knowledge Proof**: Cryptographic method proving knowledge or credential validity without revealing the underlying information
- **KYC (Know Your Customer)**: Identity verification process required for platform participation, implemented using zero-knowledge proofs
- **PII (Personally Identifiable Information)**: Sensitive personal data that is never stored on-chain in AetherLock's zkMe integration

## Storage and Data

- **IPFS (InterPlanetary File System)**: Decentralized content-addressed storage system for evidence files
- **Pinata**: IPFS pinning service ensuring persistent availability of uploaded evidence
- **Web3.Storage**: Alternative IPFS pinning service with Protocol Labs backing
- **Content Addressing**: Method of referencing data by its cryptographic hash rather than location

## Solana-Specific Terms

- **Program Derived Address (PDA)**: Deterministic Solana account addresses derived from seeds, enabling secure cross-program invocations
- **Cross-Program Invocation (CPI)**: Solana mechanism allowing one program to call another program's instructions
- **Rent**: Solana's mechanism for maintaining account state, requiring minimum balance or rent-exempt status
- **Devnet**: Solana development network for testing applications before mainnet deployment
- **Mainnet-Beta**: Solana's production network for live applications
- **SPL Token**: Solana Program Library token standard for fungible and non-fungible tokens

## ZetaChain-Specific Terms

- **Universal App**: Smart contract deployed on ZetaChain that can interact with multiple blockchain networks
- **xCall**: ZetaChain function for initiating cross-chain messages and asset transfers
- **onCall**: Handler function executed when a cross-chain message is received
- **onRevert**: Handler function executed when a cross-chain transaction fails
- **onAbort**: Handler function executed when a cross-chain transaction is aborted
- **Gateway**: ZetaChain contract managing cross-chain message routing and validation

## Cryptographic Primitives

- **Ed25519**: Elliptic curve signature scheme used for authenticating AI verification results
- **Signature Verification**: Process of validating cryptographic signatures to ensure message authenticity
- **Nonce**: Number used once to prevent replay attacks in transaction processing
- **Hash Function**: One-way cryptographic function producing fixed-size output from arbitrary input

## API and Integration

- **REST API**: Representational State Transfer API for backend service interactions
- **WebSocket**: Protocol enabling real-time bidirectional communication for live updates
- **Webhook**: HTTP callback mechanism for receiving event notifications from external services
- **Rate Limiting**: Technique for controlling API request frequency to prevent abuse
- **Chainlink Functions**: Decentralized oracle network for executing off-chain computations

## Development and Deployment

- **Environment Variables**: Configuration values stored outside code for security and flexibility
- **Docker**: Containerization platform for packaging applications with dependencies
- **Vercel**: Cloud platform for deploying frontend applications with automatic scaling
- **Render**: Cloud platform for deploying backend services with managed infrastructure
- **AWS Lambda**: Serverless compute service for running backend functions on demand

## Frontend Technologies

- **React**: JavaScript library for building user interfaces with component-based architecture
- **Next.js**: React framework with server-side rendering and static site generation
- **Wallet Adapter**: Library for integrating blockchain wallet connections into web applications
- **Phantom**: Popular Solana wallet browser extension
- **Solflare**: Multi-chain wallet supporting Solana and other networks
- **Tonkeeper**: Wallet application for TON blockchain

## Security Concepts

- **Access Control**: Mechanisms for restricting system access to authorized users and programs
- **Role-Based Access Control (RBAC)**: Security model assigning permissions based on user roles
- **Multi-Signature**: Requirement for multiple parties to approve a transaction
- **Replay Protection**: Security measure preventing reuse of valid transactions
- **Key Management**: Practices for securely storing, rotating, and handling cryptographic keys
- **Threat Model**: Analysis of potential security vulnerabilities and attack vectors

## Business and Economics

- **Revenue Model**: 10% fee structure on successful escrow transactions
- **Fee Distribution**: Allocation of collected fees between protocol treasury and stakeholders
- **Addressable Market**: Total available market size, estimated at $1.5T for freelance economy
- **Transaction Volume**: Total value of assets processed through escrow contracts
- **Profit Margin**: Percentage of revenue retained after operational costs

## User Roles

- **Client**: Party creating escrow and funding the contract for work to be completed
- **Freelancer**: Party performing work and submitting evidence for verification
- **Moderator**: Authorized party with elevated permissions for room management
- **Arbitrator**: Third party resolving disputes when AI verification is inconclusive
- **Administrator**: System operator with full access to platform management functions

## Workflow States

- **Pending**: Escrow created but not yet funded
- **Active**: Escrow funded and awaiting work submission
- **Submitted**: Evidence uploaded and awaiting AI verification
- **Approved**: AI verification successful, funds released to freelancer
- **Rejected**: AI verification failed, funds returned to client
- **Disputed**: Manual review required due to inconclusive AI results
- **Resolved**: Dispute settled through arbitration process

- **Mermaid**: Markdown-based diagramming tool for creating architecture and sequence diagrams
- **MDX**: Markdown format supporting JSX components for interactive documentation
- **Syntax Highlighting**: Code formatting feature displaying different language elements in distinct colors
- **Code Block**: Formatted section of documentation containing example code
