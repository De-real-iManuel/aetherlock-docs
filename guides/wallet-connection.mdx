# Wallet Connection Guide

AetherLock Protocol supports multiple blockchain networks, requiring different wallet connections for each chain. This guide walks you through connecting wallets for Solana, Sui, and TON networks.

## Supported Wallets

### Solana Network
- **Phantom** (Recommended)
- **Solflare**
- **Backpack**
- **Glow**

### Sui Network
- **Sui Wallet** (Official)
- **Suiet Wallet**
- **Ethos Wallet**

### TON Network
- **Tonkeeper** (Recommended)
- **TON Wallet**

## Solana Wallet Connection

### Using Phantom Wallet

<Steps>
<Step title="Install Phantom">
1. Visit [phantom.app](https://phantom.app) and download the browser extension
2. Create a new wallet or import an existing one using your seed phrase
3. Set up your password and security preferences
</Step>

<Step title="Connect to AetherLock">
1. Navigate to the AetherLock Protocol dashboard
2. Click the "Connect Wallet" button in the top right corner
3. Select "Phantom" from the wallet options
4. Approve the connection request in the Phantom popup
5. Your Solana address will appear in the dashboard
</Step>

<Step title="Switch Networks (if needed)">
1. Click the network selector in Phantom
2. Choose between:
   - **Mainnet Beta** (for production transactions)
   - **Devnet** (for testing)
   - **Testnet** (for development)
3. Refresh the AetherLock page to update the connection
</Step>
</Steps>

### Using Solflare Wallet

<Steps>
<Step title="Install Solflare">
1. Visit [solflare.com](https://solflare.com) and install the extension
2. Create or import your wallet
3. Complete the security setup
</Step>

<Step title="Connect to AetherLock">
1. On the AetherLock dashboard, click "Connect Wallet"
2. Select "Solflare" from the available options
3. Authorize the connection in the Solflare popup
4. Verify your address appears correctly
</Step>
</Steps>

## Sui Wallet Connection

### Using Sui Wallet (Official)

<Steps>
<Step title="Install Sui Wallet">

- **Mainnet** (for live transactions)
- **Testnet** (for testing)

</Steps>

## Multi-Chain Wallet Management

### Switching Between Networks

AetherLock automatically detects which network you're using based on your connected wallet:

```typescript
// Example: Network detection
const detectNetwork = () => {
  if (solanaWallet.connected) return 'solana';
  if (suiWallet.connected) return 'sui';
  if (tonWallet.connected) return 'ton';
  return null;
};
```

### Managing Multiple Wallets

You can connect wallets from different networks simultaneously:

1. **Primary Wallet**: Used for escrow creation and main transactions
2. **Secondary Wallets**: Used for cross-chain operations
3. **Backup Wallets**: For redundancy and security

<Note>
Each network requires its own native tokens for transaction fees:
- **Solana**: SOL tokens
- **Sui**: SUI tokens  
- **TON**: TON tokens
</Note>

## Troubleshooting

### Common Connection Issues

#### Wallet Not Detected

**Problem**: AetherLock doesn't show your wallet as an option

**Solutions**:
1. Refresh the browser page
2. Disable and re-enable the wallet extension
3. Clear browser cache and cookies
4. Try incognito/private browsing mode
5. Update the wallet extension to the latest version

#### Connection Fails

**Problem**: Wallet connection request fails or times out

**Solutions**:
1. Check if the wallet is unlocked
2. Ensure you're on the correct network
3. Disable other wallet extensions temporarily
4. Restart the browser
5. Check for wallet extension updates

#### Wrong Network

**Problem**: Connected to wrong blockchain network

**Solutions**:
1. Open your wallet extension
2. Click the network selector
3. Choose the correct network (Mainnet/Testnet/Devnet)
4. Refresh the AetherLock page
5. Reconnect your wallet

#### Transaction Failures

**Problem**: Transactions fail to process

**Solutions**:
1. **Insufficient Funds**: Ensure you have enough tokens for gas fees
2. **Network Congestion**: Wait and retry during off-peak hours
3. **Slippage**: Increase slippage tolerance in wallet settings
4. **Nonce Issues**: Reset account in wallet settings (advanced)

### Network-Specific Issues

#### Solana Issues

**RPC Errors**:
```bash
# Switch to a different RPC endpoint
Mainnet: https://api.mainnet-beta.solana.com
Devnet: https://api.devnet.solana.com
```

**Program Errors**:
- Check if the AetherLock program is deployed on your network
- Verify account permissions and signatures

#### Sui Issues

**Gas Estimation Failures**:
- Ensure sufficient SUI balance for gas
- Try reducing transaction complexity

**Object Ownership**:
- Verify you own the objects being used in transactions
- Check for object locks or pending transactions

#### TON Issues

**Wallet Sync**:
- Wait for wallet to fully sync with TON network
- Check network status at [tonscan.org](https://tonscan.org)

**Smart Contract Interaction**:
- Verify contract addresses are correct
- Check if contracts are deployed on the network you're using

## Security Best Practices

### Wallet Security

<Warning>
Never share your seed phrase or private keys with anyone. AetherLock will never ask for this information.
</Warning>

1. **Use Hardware Wallets**: For large amounts, consider Ledger or Trezor
2. **Enable 2FA**: Where available, enable two-factor authentication
3. **Regular Backups**: Keep multiple secure copies of seed phrases
4. **Software Updates**: Keep wallet software updated
5. **Phishing Protection**: Always verify URLs before connecting

### Transaction Security

1. **Verify Addresses**: Double-check recipient addresses
2. **Check Amounts**: Confirm transaction amounts before signing
3. **Review Permissions**: Understand what you're authorizing
4. **Use Test Networks**: Test with small amounts first
5. **Monitor Activity**: Regularly check transaction history

### Network Security

1. **Use HTTPS**: Always use secure connections
2. **Avoid Public WiFi**: Don't transact on public networks
3. **VPN Usage**: Consider using a VPN for additional privacy
4. **Browser Security**: Keep browsers updated and use reputable extensions
